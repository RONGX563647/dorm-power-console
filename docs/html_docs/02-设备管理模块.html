二、需求分析与设计

2.1 业务场景

场景 1：设备注册

新设备到货 → 录入系统 → 分配房间 → 设备上线 → 状态变为 ONLINE

场景 2：设备监控

设备定时上报数据 → MQTT Broker → MqttBridge → 更新设备状态 → WebSocket 推送前端

场景 3：设备分组

创建设备组 → 添加设备到组 → 批量操作组内设备 → 查看组统计

场景 4：固件升级

上传固件 → 选择目标设备 → 推送升级命令 → 设备下载升级 → 上报新版本

2.3 技术选型与理由

技术 说明 版本 选型理由
MQTT (Eclipse Paho) IoT 通信协议 1.3.x ✅ 轻量级：协议头最小仅 2 字节，适合低带宽环境
✅ 发布/订阅：解耦设备与服务端，支持一对多通信
✅ QoS 保障：支持 3 个等级（0/1/2）的消息可靠性保障
✅ 遗嘱消息：设备异常离线时可发送遗嘱消息
✅ 低功耗：适合电池供电的 IoT 设备
Spring Data JPA ORM 框架 3.2.x ✅ 开发效率：注解驱动，减少样板代码
✅ 数据库无关：可轻松切换数据库
✅ 级联操作：自动处理设备与关联实体关系
⚠️ 注意：实际项目使用 Long/IDENTITY 而非 UUID，便于数据库自增
WebSocket 实时推送 - ✅ 双向通信：服务端可主动推送设备状态到前端
✅ 低延迟：保持长连接，避免轮询开销
✅ 标准协议：浏览器原生支持，无需额外插件
Lombok 代码简化 - ✅ 减少样板：自动生成 Getter/Setter/Constructor
✅ 可读性：减少视觉干扰，聚焦业务逻辑
✅ 维护性：减少手动生成代码的错误

技术对比与取舍： - MQTT vs HTTP：选择 MQTT 因为 IoT 设备需要长连接、低功耗、实时推送 - JPA vs MyBatis：选择 JPA 因为设备管理 CRUD 操作多，JPA 开发效率更高 - WebSocket vs SSE：选择 WebSocket 因为需要双向通信（前端也可发送控制命令）

package com.dormpower.model;

import jakarta.persistence.*;
import lombok.Data;
import java.time.LocalDateTime;
import java.util.UUID;

@Data
@Entity
@Table(name = "device")
public class Device {

    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    private UUID id;

    @Column(unique = true, nullable = false, length = 50)
    private String deviceId; // 设备唯一标识（如 MAC 地址）

    @Column(nullable = false, length = 100)
    private String name; // 设备名称

    private String description;

    @Enumerated(EnumType.STRING)
    private DeviceType type; // 设备类型

    @Column(length = 50)
    private String model; // 设备型号

    @Column(length = 50)
    private String firmwareVersion; // 固件版本

    @Enumerated(EnumType.STRING)
    private DeviceStatus status = DeviceStatus.OFFLINE; // 设备状态

    @Column(length = 50)
    private String ipAddress;

    @Column(length = 50)
    private String macAddress;

    private LocalDateTime lastOnlineAt;
    private LocalDateTime lastOfflineAt;

    private LocalDateTime createdAt = LocalDateTime.now();
    private LocalDateTime updatedAt = LocalDateTime.now();

    @Column(name = "room_id")
    private UUID roomId; // 关联房间

    @Column(name = "group_id")
    private UUID groupId; // 关联设备组

    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }
}

enum DeviceType {
    SOCKET,    // 智能插座
    STRIP,     // 智能排插
    METER      // 智能电表
}

enum DeviceStatus {
    ONLINE,    // 在线
    OFFLINE,   // 离线
    FAULT      // 故障
}

1.4 创建 DTO 数据传输对象

为什么需要 DTO? - 分离内部模型和外部接口 - 控制数据暴露范围(不暴露内部字段) - 添加数据验证注解 - 支持 API 版本控制

请求 DTO 设计:

文件位置: backend/src/main/java/com/dormpower/dto/DeviceRegistrationRequest.java

package com.dormpower.dto;

import io.swagger.v3.oas.annotations.media.Schema;
import jakarta.validation.constraints.NotBlank;
import jakarta.validation.constraints.Pattern;
import jakarta.validation.constraints.Size;

/**
 * 设备注册请求DTO
 *
 * 用于设备自动注册时接收设备信息，包含参数校验规则。
 *
 * 校验规则：
 * - 设备ID：只能包含字母、数字、下划线，长度不超过64字符
 * - 设备名称：不能为空，长度不超过100字符
 * - 房间号：格式为 楼栋号-房间号（如：A1-301）
 *
 * @author dormpower team
 * @version 1.0
 */
@Schema(description = "设备注册请求")
public class DeviceRegistrationRequest {

    @NotBlank(message = "设备ID不能为空")
    @Pattern(regexp = "^[a-zA-Z0-9_]{1,64}$", message = "设备ID只能包含字母、数字、下划线，长度不超过64字符")
    @Schema(description = "设备ID", example = "device_001", required = true)
    private String id;

    @NotBlank(message = "设备名称不能为空")
    @Size(max = 100, message = "设备名称长度不能超过100字符")
    @Schema(description = "设备名称", example = "A1-301智能插座", required = true)
    private String name;

    @NotBlank(message = "房间号不能为空")
    @Pattern(regexp = "^[A-Za-z]\\d{1,2}-\\d{3,4}$", message = "房间格式错误，正确格式：A1-301")
    @Schema(description = "房间号", example = "A1-301", required = true)
    private String room;

    public String getId() {
        return id;
    }

    public void setId(String id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getRoom() {
        return room;
    }

    public void setRoom(String room) {
        this.room = room;
    }
}

DTO 设计原则:

┌─────────────────────────────────────────────┐
│            DTO 设计最佳实践                   │
├─────────────────────────────────────────────┤
│ 1. 职责单一:                                 │
│    - Request DTO: 接收前端数据,添加验证       │
│    - Response DTO: 返回前端数据,过滤敏感信息  │
│    - Message DTO: 消息传递载体(MQTT/WebSocket)│
│                                            │
│ 2. 命名规范:                                 │
│    - 请求DTO: XxxRequest.java                │
│    - 响应DTO: XxxResponse.java               │
│    - 消息DTO: XxxMessage.java                │
│                                            │
│ 3. 验证注解:                                 │
│    - @NotNull: 不能为null                    │
│    - @NotBlank: 不能为空白字符串              │
│    - @Size: 字符串/集合大小限制               │
│    - @Pattern: 正则表达式验证                 │
│    - @Min/@Max: 数值范围验证                 │
│                                            │
│ 4. API文档:                                  │
│    - @Schema: OpenAPI 3.0 文档注解           │
│    - description: 字段说明                   │
│    - example: 示例值                         │
│    - required: 是否必填                      │
│                                            │
│ 5. 与Entity的区别:                           │
│    - Entity: 数据库映射,包含JPA注解           │
│    - DTO: 数据传输,包含验证注解               │
│    - 两者可以相互转换,使用Mapper或手动转换    │
│                                            │
│ 6. Builder模式:                              │
│    - 响应DTO推荐使用Builder模式               │
│    - 链式调用,代码更简洁                      │
│    - 避免构造函数参数过多                     │
└─────────────────────────────────────────────┘

3.2 DeviceServiceImpl 实现

文件位置: backend/src/main/java/com/dormpower/service/impl/DeviceServiceImpl.java

package com.dormpower.service.impl;

import com.dormpower.dto.request.CreateDeviceRequest;
import com.dormpower.dto.response.DeviceResponse;
import com.dormpower.model.Device;
import com.dormpower.model.DeviceStatus;
import com.dormpower.repository.DeviceRepository;
import com.dormpower.service.DeviceService;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;
import java.util.UUID;
import java.util.stream.Collectors;

@Service
@RequiredArgsConstructor
public class DeviceServiceImpl implements DeviceService {

    private final DeviceRepository deviceRepository;

    @Override
    @Transactional
    public DeviceResponse createDevice(CreateDeviceRequest request) {
        // 检查设备 ID 是否重复
        if (deviceRepository.existsByDeviceId(request.getDeviceId())) {
            throw new RuntimeException("设备 ID 已存在");
        }

        Device device = new Device();
        device.setDeviceId(request.getDeviceId());
        device.setName(request.getName());
        device.setDescription(request.getDescription());
        device.setType(request.getType());
        device.setModel(request.getModel());
        device.setRoomId(request.getRoomId());
        device.setGroupId(request.getGroupId());
        device.setStatus(DeviceStatus.OFFLINE);

        Device saved = deviceRepository.save(device);
        return toResponse(saved);
    }

    @Override
    public DeviceResponse getDevice(UUID id) {
        Device device = deviceRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("设备不存在"));
        return toResponse(device);
    }

    @Override
    public List<DeviceResponse> getDevices(UUID roomId, UUID groupId, DeviceStatus status) {
        List<Device> devices;

        if (roomId != null) {
            devices = deviceRepository.findByRoomId(roomId);
        } else if (groupId != null) {
            devices = deviceRepository.findByGroupId(groupId);
        } else if (status != null) {
            devices = deviceRepository.findByStatus(status);
        } else {
            devices = deviceRepository.findAll();
        }

        return devices.stream().map(this::toResponse).collect(Collectors.toList());
    }

    @Override
    @Transactional
    public DeviceResponse updateDevice(UUID id, CreateDeviceRequest request) {
        Device device = deviceRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("设备不存在"));

        device.setName(request.getName());
        device.setDescription(request.getDescription());
        device.setRoomId(request.getRoomId());
        device.setGroupId(request.getGroupId());

        Device updated = deviceRepository.save(device);
        return toResponse(updated);
    }

    @Override
    @Transactional
    public void deleteDevice(UUID id) {
        deviceRepository.deleteById(id);
    }

    @Override
    @Transactional
    public void updateDeviceStatus(String deviceId, DeviceStatus status) {
        Device device = deviceRepository.findByDeviceId(deviceId)
            .orElseThrow(() -> new RuntimeException("设备不存在"));

        DeviceStatus oldStatus = device.getStatus();
        device.setStatus(status);

        if (status == DeviceStatus.ONLINE) {
            device.setLastOnlineAt(java.time.LocalDateTime.now());
        } else if (status == DeviceStatus.OFFLINE) {
            device.setLastOfflineAt(java.time.LocalDateTime.now());
        }

        deviceRepository.save(device);

        // TODO: 记录状态变更历史
    }

    private DeviceResponse toResponse(Device device) {
        DeviceResponse response = new DeviceResponse();
        response.setId(device.getId());
        response.setDeviceId(device.getDeviceId());
        response.setName(device.getName());
        response.setDescription(device.getDescription());
        response.setType(device.getType());
        response.setModel(device.getModel());
        response.setFirmwareVersion(device.getFirmwareVersion());
        response.setStatus(device.getStatus());
        response.setRoomId(device.getRoomId());
        response.setGroupId(device.getGroupId());
        response.setCreatedAt(device.getCreatedAt());
        return response;
    }
}

import { defineStore } from 'pinia';
import { ref } from 'vue';
import type { Device, DeviceGroup } from '@/types/device';
import { deviceApi, deviceGroupApi } from '@/api/device';

export const useDeviceStore = defineStore('device', () => {
  const devices = ref<Device[]>([]);
  const groups = ref<DeviceGroup[]>([]);
  const loading = ref(false);

  // 加载设备列表
  async function loadDevices(params?: any) {
    loading.value = true;
    try {
      devices.value = await deviceApi.getDevices(params);
    } finally {
      loading.value = false;
    }
  }

  // 加载设备组
  async function loadGroups() {
    groups.value = await deviceGroupApi.getGroups();
  }

  // 添加设备
  async function addDevice(device: Device) {
    devices.value.push(device);
  }

  // 更新设备
  function updateDevice(id: string, updates: Partial<Device>) {
    const index = devices.value.findIndex(d => d.id === id);
    if (index !== -1) {
      devices.value[index] = { ...devices.value[index], ...updates };
    }
  }

  // 删除设备
  function removeDevice(id: string) {
    const index = devices.value.findIndex(d => d.id === id);
    if (index !== -1) {
      devices.value.splice(index, 1);
    }
  }

  return {
    devices,
    groups,
    loading,
    loadDevices,
    loadGroups,
    addDevice,
    updateDevice,
    removeDevice
  };
});
