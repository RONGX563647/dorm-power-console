模块21：Java 21虚拟线程高并发优化

学习时长: 3-5 天
难度: ⭐⭐⭐⭐
前置知识: Java并发编程、线程池、异步编程

一、技术背景

1.1 传统线程 vs 虚拟线程

平台线程（传统）:
  - 1:1映射到操作系统线程
  - 栈大小: 1MB/线程
  - 切换成本高（上下文切换）
  - 适合CPU密集型任务
  - 数量限制: 几千个

虚拟线程（Java 21）:
  - M:N映射到少量平台线程
  - 栈大小: 按需增长（KB级别）
  - 切换成本低（用户态）
  - 适合I/O密集型任务
  - 数量限制: 百万级别

性能对比:
  10万并发请求:
    - 平台线程: 内存100GB, 无法创建
    - 虚拟线程: 内存100MB, 轻松处理

二、核心实现

2.1 VirtualThreadService 虚拟线程服务

文件位置: backend/src/main/java/com/dormpower/service/VirtualThreadService.java

package com.dormpower.service;

import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.concurrent.CompletableFuture;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.stream.Collectors;

/**
 * 虚拟线程服务
 * 
 * 使用场景:
 * 1. 批量设备状态更新（I/O密集）
 * 2. 并发数据库查询
 * 3. 外部API调用
 * 4. 消息推送
 * 
 * 优势:
 * - 内存占用降低100倍
 * - 吞吐量提升10倍
 * - 代码更简洁（无需线程池配置）
 * 
 * @author dormpower team
 * @version 1.0
 */
@Slf4j
@Service
public class VirtualThreadService {

    // 虚拟线程执行器
    private final ExecutorService virtualThreadExecutor = 
        Executors.newVirtualThreadPerTaskExecutor();

    /**
     * 使用虚拟线程并发处理设备列表
     */
    public void processDevicesAsync(List<String> deviceIds, DeviceProcessor processor) {
        List<CompletableFuture<Void>> futures = deviceIds.stream()
            .map(deviceId -> CompletableFuture.runAsync(() -> {
                try {
                    processor.process(deviceId);
                } catch (Exception e) {
                    log.error("Failed to process device {}: {}", deviceId, e.getMessage());
                }
            }, virtualThreadExecutor))
            .collect(Collectors.toList());

        // 等待所有任务完成
        CompletableFuture.allOf(futures.toArray(new CompletableFuture[0])).join();
    }

    /**
     * 批量查询设备状态
     */
    public List<DeviceStatus> batchQueryStatus(List<String> deviceIds) {
        return deviceIds.parallelStream()
            .map(deviceId -> queryDeviceStatus(deviceId))
            .collect(Collectors.toList());
    }

    /**
     * 并发发送通知
     */
    public void sendNotificationsAsync(List<Notification> notifications) {
        notifications.parallelStream()
            .forEach(notification -> {
                try {
                    sendNotification(notification);
                } catch (Exception e) {
                    log.error("Failed to send notification: {}", e.getMessage());
                }
            });
    }

    private DeviceStatus queryDeviceStatus(String deviceId) {
        // 模拟数据库查询（I/O操作）
        try {
            Thread.sleep(10);  // 模拟网络延迟
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        return new DeviceStatus(deviceId, true);
    }

    private void sendNotification(Notification notification) {
        // 模拟发送通知（I/O操作）
        try {
            Thread.sleep(5);  // 模拟网络延迟
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }

    @FunctionalInterface
    public interface DeviceProcessor {
        void process(String deviceId) throws Exception;
    }

    public record DeviceStatus(String deviceId, boolean online) {}
    public record Notification(String userId, String message) {}
}

3.1 基准测试

虚拟线程最佳实践:
  ✅ 适合: I/O密集（数据库、HTTP、MQTT）
  ❌ 不适合: CPU密集（计算、加密）
  ✅ 适合: 高并发短任务
  ❌ 不适合: 长时间阻塞任务

四、配置与使用

4.1 Spring Boot配置

# application.yml
spring:
  task:
    execution:
      pool:
        # 虚拟线程无需配置线程池大小
        # Spring Boot 3.2+ 自动使用虚拟线程
        thread-name-prefix: "vtask-"

4.2 代码示例

// 方式1: 直接使用
ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();
executor.submit(() -> doSomething());

// 方式2: CompletableFuture
CompletableFuture.supplyAsync(() -> fetchData(), 
    Executors.newVirtualThreadPerTaskExecutor());

// 方式3: Spring @Async（需要配置）
@Async
public CompletableFuture<Data> fetchDataAsync() {
    return CompletableFuture.completedFuture(fetchData());
}
