二、需求分析与设计

2.1 业务场景

场景 1：设备过载告警 设备上报电流数据 → 规则引擎检测 → 超过阈值 → 触发告警 → 推送通知 → 生成工单 → 处理闭环

场景 2：欠费告警 房间余额检查 → 低于阈值 → 触发告警 → 发送账单提醒 → 用户充值 → 告警消除

场景 3：设备离线告警 设备心跳检测 → 超时未上报 → 判定离线 → 触发告警 → 通知管理员 → 维修处理

场景 4：用电异常告警 采集用电数据 → AI 异常检测 → 发现异常模式 → 触发告警 → 安全预警

2.2 功能需求清单

功能 描述 优先级
告警规则配置 支持阈值、规则表达式 P0
实时告警检测 流式数据检测 P0
告警通知 多渠道推送 P0
告警工单 工单流转处理 P1
告警统计 报表与趋势分析 P1
告警升级 超时未处理升级 P2
告警抑制 避免告警风暴 P2
告警关联 关联告警分析 P3

2.3 技术选型与理由

技术 说明 版本 选型理由
Drools 规则引擎 8.x ✅ 规则与代码分离：业务规则配置化，无需修改代码
✅ 灵活表达：DRL 语言支持复杂规则表达
✅ 性能优秀：Rete 算法高效匹配规则
⚠️ 学习曲线：需要学习 DRL 语法，教学场景简化实现
Spring Event 事件驱动机制 3.2.x ✅ 解耦组件：告警触发与处理解耦
✅ 异步处理：支持@EventListener 异步监听
✅ 内置支持：Spring 框架自带，无需额外依赖
WebSocket 实时推送 - ✅ 实时通知：告警产生立即推送到前端
✅ 多端推送：支持多个客户端同时接收
✅ 双向通信：前端可发送告警确认/处理指令
ECharts 统计图表 5.x ✅ 告警趋势：折线图展示告警趋势
✅ 告警分布：饼图展示告警类型分布
✅ 实时监控：动态刷新告警数据

技术对比与取舍： - 规则引擎 vs if-else：简单场景用 if-else，复杂业务用规则引擎便于维护 - 轮询 vs WebSocket：选择 WebSocket 因为告警需要实时推送，轮询延迟高 - 数据库 vs 时序库：教学用数据库简化，生产环境考虑 InfluxDB 存储告警数据

import jakarta.persistence.*; import lombok.Data; import java.math.BigDecimal; import java.time.LocalDateTime; import java.util.UUID;

@Data @Entity @Table(name = "alarm_rule") public class AlarmRule {

@Id
@GeneratedValue(strategy = GenerationType.UUID)
private UUID id;

@Column(nullable = false, length = 100)
private String ruleName;

@Enumerated(EnumType.STRING)
@Column(nullable = false)
private AlarmType alarmType;

@Enumerated(EnumType.STRING)
@Column(nullable = false)
private AlarmSeverity severity = AlarmSeverity.WARNING;

@Column(length = 500)
private String ruleExpression; // 规则表达式

@Column(precision = 10, scale = 2)
private BigDecimal threshold; // 阈值

@Column(length = 50)
private String metricName; // 指标名称

private Integer durationSeconds; // 持续时间（秒）

@Column(length = 500)
private String notificationTemplate; // 通知模板

@ElementCollection
@CollectionTable(name = "alarm_rule_recipients")
@Column(name = "recipient")
private java.util.List<String> recipients; // 通知接收人

@Column(nullable = false)
private boolean enabled = true;

@Column(length = 500)
private String description;

private LocalDateTime createdAt = LocalDateTime.now();
private LocalDateTime updatedAt = LocalDateTime.now();

@PreUpdate
protected void onUpdate() {
    updatedAt = LocalDateTime.now();
}

}

enum AlarmType { OVERLOAD, // 过载 UNDER_VOLTAGE, // 欠压 OVER_VOLTAGE, // 过压 OVER_CURRENT, // 过流 OVER_USE, // 超量用电 DEVICE_OFFLINE, // 设备离线 LOW_BALANCE, // 余额不足 TEMPERATURE, // 温度异常 CUSTOM // 自定义 }

enum AlarmSeverity { INFO, // 提示 WARNING, // 警告 CRITICAL, // 严重 EMERGENCY // 紧急 } ```

```java package com.dormpower.model;

import jakarta.persistence.*; import lombok.Data; import java.time.LocalDateTime; import java.util.UUID;

@Data @Entity @Table(name = "alarm_record", indexes = { @Index(name = "idx_alarm_device_time", columnList = "deviceId, createdAt"), @Index(name = "idx_alarm_status", columnList = "status") }) public class AlarmRecord {

@Id
@GeneratedValue(strategy = GenerationType.UUID)
private UUID id;

@Column(nullable = false, length = 100)
private String alarmNumber; // 告警编号

@Column(name = "rule_id", nullable = false)
private UUID ruleId;

@Enumerated(EnumType.STRING)
@Column(nullable = false)
private AlarmType alarmType;

@Enumerated(EnumType.STRING)
@Column(nullable = false)
private AlarmSeverity severity;

@Column(name = "device_id")
private UUID deviceId;

@Column(name = "room_id")
private UUID roomId;

@Column(length = 500)
private String alarmMessage; // 告警内容

@Column(precision = 10, scale = 2)
private java.math.BigDecimal alarmValue; // 告警值

@Enumerated(EnumType.STRING)
@Column(nullable = false)
private AlarmStatus status = AlarmStatus.ACTIVE;

private LocalDateTime occurredAt; // 发生时间
private LocalDateTime acknowledgedAt; // 确认时间
private LocalDateTime resolvedAt; // 解决时间

@Column(name = "acknowledged_by")
private UUID acknowledgedBy; // 确认人

@Column(name = "resolved_by")
private UUID resolvedBy; // 解决人

@Column(length = 500)
private String resolutionNote; // 解决备注

private Integer escalateLevel = 0; // 升级级别

@Column(length = 500)
private String notificationChannels; // 通知渠道

private LocalDateTime createdAt = LocalDateTime.now();

@PrePersist
protected void onCreate() {
    if (alarmNumber == null) {
        alarmNumber = "ALM-" + LocalDateTime.now().format(
            java.time.format.DateTimeFormatter.ofPattern("yyyyMMddHHmmss")) + 
            "-" + UUID.randomUUID().toString().substring(0, 4).toUpperCase();
    }
}

}

enum AlarmStatus { ACTIVE, // 活动中 ACKNOWLEDGED, // 已确认 RESOLVED, // 已解决 SUPPRESSED, // 已抑制 CLOSED // 已关闭 } ```

文件位置: backend/src/main/java/com/dormpower/controller/AlarmController.java

```java package com.dormpower.controller;

import com.dormpower.dto.AlarmTriggerRequest; import com.dormpower.model.AlarmRecord; import com.dormpower.service.AlarmService; import lombok.RequiredArgsConstructor; import org.springframework.data.domain.Page; import org.springframework.data.domain.Pageable; import org.springframework.format.annotation.DateTimeFormat; import org.springframework.http.ResponseEntity; import org.springframework.security.access.prepost.PreAuthorize; import org.springframework.web.bind.annotation.*;

import java.time.LocalDateTime; import java.util.List; import java.util.UUID;

@RestController @RequestMapping("/api/alarms") @RequiredArgsConstructor public class AlarmController {

private final AlarmService alarmService;

@PostMapping("/trigger")
@PreAuthorize("hasAnyAuthority('alarm:write', 'admin')")
public ResponseEntity<AlarmRecord> triggerAlarm(@RequestBody AlarmTriggerRequest request) {
    return ResponseEntity.ok(alarmService.triggerAlarm(request));
}

@GetMapping
@PreAuthorize("hasAnyAuthority('alarm:read', 'admin')")
public ResponseEntity<Page<AlarmRecord>> getAlarms(
    @RequestParam(required = false) UUID deviceId,
    @RequestParam(required = false) String status,
    @RequestParam(required = false) @DateTimeFormat(iso = DateTimeFormat.ISO.DATE_TIME) LocalDateTime startTime,
    @RequestParam(required = false) @DateTimeFormat(iso = DateTimeFormat.ISO.DATE_TIME) LocalDateTime endTime,
    Pageable pageable
) {
    return ResponseEntity.ok(
        alarmService.getAlarms(deviceId, status, startTime, endTime, pageable)
    );
}

@PostMapping("/{id}/acknowledge")
@PreAuthorize("hasAnyAuthority('alarm:write', 'admin')")
public ResponseEntity<AlarmRecord> acknowledgeAlarm(
    @PathVariable UUID id,
    @RequestParam UUID userId
) {
    return ResponseEntity.ok(alarmService.acknowledgeAlarm(id, userId));
}

@PostMapping("/{id}/resolve")
@PreAuthorize("hasAnyAuthority('alarm:write', 'admin')")
public ResponseEntity<AlarmRecord> resolveAlarm(
    @PathVariable UUID id,
    @RequestParam UUID userId,
    @RequestParam(required = false) String note
) {
    return ResponseEntity.ok(alarmService.resolveAlarm(id, userId, note));
}

@GetMapping("/rules")
@PreAuthorize("hasAnyAuthority('alarm:read', 'admin')")
public ResponseEntity<List<AlarmRecord>> getActiveRules() {
    return ResponseEntity.ok((List) alarmService.getActiveRules());
}

} ```

6.2 常见问题与解决方案

问题 1：告警风暴

现象: 短时间内产生大量告警

解决方案: ```java // 告警抑制 public class AlarmSuppressor { private Map lastAlarmTime = new ConcurrentHashMap<>(); private static final long SUPPRESS_WINDOW_MS = 300000; // 5 分钟

public boolean shouldSuppress(String alarmKey) {
    LocalDateTime lastTime = lastAlarmTime.get(alarmKey);
    if (lastTime != null && 
        System.currentTimeMillis() - lastTime.atTime(java.time.LocalTime.now()).toInstant(
            java.time.ZoneOffset.UTC).toEpochMilli() < SUPPRESS_WINDOW_MS) {
        return true;
    }
    lastAlarmTime.put(alarmKey, LocalDateTime.now());
    return false;
}

} ```
