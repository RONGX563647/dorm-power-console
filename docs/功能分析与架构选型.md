# 宿舍电源管理系统 - 功能分析与架构选型文档

## 1. 文档概述

### 1.1 文档目的

本文档旨在对宿舍电源管理系统进行全面的功能分析与架构选型说明，帮助开发团队、项目管理人员和相关利益方理解系统的功能需求、技术选型依据和架构设计思路。

### 1.2 适用范围

本文档适用于宿舍电源管理系统的设计、开发、测试和运维阶段，涵盖后端服务、前端应用、移动端应用和物联网设备通信等各个方面。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| IoT | 物联网（Internet of Things），指通过互联网连接物理设备实现数据交换和控制 |
| MQTT | 消息队列遥测传输协议，轻量级发布/订阅消息传输协议 |
| WebSocket | 全双工通信协议，支持客户端与服务器之间实时双向通信 |
| JWT | JSON Web Token，用于身份认证的无状态令牌 |
| RBAC | 基于角色的访问控制（Role-Based Access Control） |
| JPA | Java持久化API，用于对象关系映射（ORM） |
| DDD | 领域驱动设计（Domain-Driven Design） |

#### 2.3.3.1 身份认证设计与实现

**设计目标**：实现无状态的JWT令牌认证，支持多终端登录，确保持久安全性。

**技术选型**：
- JWT库：jjwt (0.12.5)
- 签名算法：HS512 (HMAC-SHA512)

**JWT令牌结构**：
```java
// 载荷声明
Claims claims = Jwts.claims()
    .subject(username)
    .issuedAt(new Date())
    .expiration(new Date(System.currentTimeMillis() + 24*60*60*1000))
    .add("roles", roles)
    .add("deviceId", deviceId)
    .build();

// 密钥生成（建议使用至少512位的密钥）
Key key = Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
```

**令牌管理策略**：
| 策略项 | 配置值 | 说明 |
|--------|--------|------|
| 令牌有效期 | 24小时 | JWT过期时间 |
| 刷新令牌有效期 | 7天 | 长期访问授权 |
| 令牌存储 | HttpOnly Cookie | 防止XSS攻击 |
| 签名算法 | HS512 | 对称加密算法 |
| 密钥长度 | 512位 | 安全强度要求 |

**认证流程**：
```
1. 用户登录 → 验证用户名密码 → 生成JWT令牌 → 返回令牌
2. 访问受保护资源 → 提取Authorization头 → 验证JWT签名 → 解析载荷 → 检查权限 → 处理请求
3. 令牌过期 → 返回401 → 提示重新登录或使用刷新令牌
```

**核心代码实现**：
```java
@Component
public class JwtTokenProvider {
    private final Key key;
    private final long validityInMs = 24 * 60 * 60 * 1000; // 24小时
    
    public String createToken(String username, List<String> roles) {
        Claims claims = Jwts.claims()
            .subject(username)
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + validityInMs))
            .add("roles", roles)
            .build();
        
        return Jwts.builder()
            .claims(claims)
            .signWith(key, Jwts.SIG.HS512)
            .compact();
    }
    
    public boolean validateToken(String token) {
        try {
            Jwts.parser().verifyWith(key).build().parseSignedClaims(token);
            return true;
        } catch (JwtException | IllegalArgumentException e) {
            return false;
        }
    }
}
```

#### 2.3.3.2 密码安全设计与实现

**设计目标**：使用PBKDF2算法加密存储用户密码，防范彩虹表攻击和暴力破解。

**技术选型**：
- 加密算法：PBKDF2WithHmacSHA512
- 盐值长度：32字节
- 迭代次数：310000次（OWASP推荐值）
- 密钥长度：512位

**密码存储策略**：
| 策略项 | 配置值 | 说明 |
|--------|--------|------|
| 哈希算法 | PBKDF2WithHmacSHA512 | 慢哈希算法 |
| 盐值长度 | 32字节 | 256位随机盐 |
| 迭代次数 | 310000 | OWASP 2023推荐 |
| 密钥长度 | 512位 | 输出哈希长度 |
| 密码最小长度 | 8位 | 防止弱密码 |

**密码验证流程**：
```
用户输入密码 → 读取数据库存储的盐值 → 使用相同算法迭代计算 → 比对哈希值 → 返回验证结果
```

**核心代码实现**：
```java
@Component
public class PasswordEncoder {
    private static final int SALT_LENGTH = 32;
    private static final int ITERATION_COUNT = 310000;
    private static final int KEY_LENGTH = 512;
    
    public String encode(String rawPassword) {
        byte[] salt = new byte[SALT_LENGTH];
        new SecureRandom().nextBytes(salt);
        
        PBEKeySpec spec = new PBEKeySpec(
            rawPassword.toCharArray(), 
            salt, 
            ITERATION_COUNT, 
            KEY_LENGTH
        );
        
        SecretKeyFactory factory = SecretKeyFactory.getInstance("PBKDF2WithHmacSHA512");
        byte[] hash = factory.generateSecret(spec).getEncoded();
        
        // 存储格式：盐值:哈希值 (Base64编码)
        return Base64.getEncoder().encodeToString(salt) + ":" + 
               Base64.getEncoder().encodeToString(hash);
    }
    
    public boolean matches(String rawPassword, String encodedPassword) {
        String[] parts = encodedPassword.split(":");
        byte[] salt = Base64.getDecoder().decode(parts[0]);
        byte[] storedHash = Base64.getDecoder().decode(parts[1]);
        
        PBEKeySpec spec = new PBEKeySpec(
            rawPassword.toCharArray(), 
            salt, 
            ITERATION_COUNT, 
            KEY_LENGTH
        );
        
        SecretKeyFactory factory = SecretKeyFactory.getInstance("PBKDF2WithHmacSHA512");
        byte[] computedHash = factory.generateSecret(spec).getEncoded();
        
        return Arrays.equals(storedHash, computedHash);
    }
}
```

#### 2.3.3.3 传输安全设计与实现

**设计目标**：全链路HTTPS/WSS加密，确保数据在传输过程中不被窃听或篡改。

**SSL/TLS配置**：
| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| TLS版本 | TLS 1.3 | 最高安全版本 |
| 密码套件 | TLS_AES_256_GCM_SHA384 | 强加密算法 |
| 证书类型 | Let's Encrypt | 免费SSL证书 |
| 证书自动续期 | 30天 | 避免证书过期 |

**Nginx SSL配置**：
```nginx
server {
    listen 443 ssl http2;
    server_name api.dorm.com;
    
    ssl_certificate /etc/letsencrypt/live/api.dorm.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.dorm.com/privkey.pem;
    
    # TLS配置
    ssl_protocols TLSv1.3;
    ssl_ciphers TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256;
    ssl_prefer_server_ciphers on;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_session_tickets off;
    
    # HSTS配置
    add_header Strict-Transport-Security "max-age=63072000" always;
    
    location / {
        proxy_pass http://backend:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# HTTP重定向到HTTPS

    return 301 https://$server_name$request_uri;
}
```

#### 2.3.3.5 接口限流设计与实现

**设计目标**：使用令牌桶算法实现接口限流，防止API滥用和DDoS攻击。

**限流策略配置**：
| 接口类型 | 限流阈值 | 限流算法 | 说明 |
|----------|----------|----------|------|
| 登录接口 | 5次/分钟/IP | 令牌桶 | 防止暴力破解 |
| 注册接口 | 3次/分钟/IP | 令牌桶 | 防止恶意注册 |
| 查询接口 | 60次/分钟/用户 | 令牌桶 | 正常用户访问 |
| 控制接口 | 30次/分钟/用户 | 令牌桶 | 保护设备稳定性 |
| 文件上传 | 10次/分钟/用户 | 令牌桶 | 防止资源耗尽 |

**令牌桶算法实现**：
```java
@Component
public class RateLimiter {
    private final RedisTemplate<String, String> redisTemplate;
    private static final String RATE_LIMIT_PREFIX = "rate:limit:";
    
    public boolean tryAcquire(String key, int permits, int windowSeconds) {
        String redisKey = RATE_LIMIT_PREFIX + key;
        Long count = redisTemplate.opsForValue().increment(redisKey);
        
        if (count == 1) {
            redisTemplate.expire(redisKey, windowSeconds, TimeUnit.SECONDS);
        }
        
        return count <= permits;
    }
}

// 分布式限流注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RateLimit {
    int permits() default 60;
    int window() default 60;
    String key() default "";
}

// 限流切面实现
@Aspect
@Component
public class RateLimitAspect {
    private final RateLimiter rateLimiter;
    
    @Around("@annotation(rateLimit)")
    public Object around(ProceedingJoinPoint joinPoint, RateLimit rateLimit) throws Throwable {
        String key = getKey(joinPoint, rateLimit);
        
        if (!rateLimiter.tryAcquire(key, rateLimit.permits(), rateLimit.window())) {
            throw new RateLimitExceededException("请求过于频繁，请稍后重试");
        }
        
        return joinPoint.proceed();
    }
}
```

```java
@Aspect
@Component
@RequiredArgsConstructor
public class AuditAspect {
    private final AuditLogRepository auditLogRepository;
    
    @Pointcut("@annotation(org.springframework.web.bind.annotation.*)")
    public void controllerPointcut() {}
    
    @Around("controllerPointcut() && @annotation(requestMapping)")
    public Object around(ProceedingJoinPoint joinPoint, RequestMapping requestMapping) 
        throws Throwable {
        
        HttpServletRequest request = getRequest();
        long startTime = System.currentTimeMillis();
        
        AuditLog log = new AuditLog();
        log.setId(UUID.randomUUID().toString());
        log.setUsername(getCurrentUsername());
        log.setOperation(getOperation(request));
        log.setResource(getResource(request));
        log.setMethod(request.getMethod());
        log.setParams(getParams(joinPoint));
        log.setIp(getClientIp(request));
        log.setUserAgent(request.getHeader("User-Agent"));
        log.setTimestamp(LocalDateTime.now());
        
        Object result = null;
        try {
            result = joinPoint.proceed();
            log.setResult("SUCCESS");
        } catch (Exception e) {
            log.setResult("FAILED: " + e.getMessage());
            throw e;
        } finally {
            log.setDuration(System.currentTimeMillis() - startTime);
            auditLogRepository.save(log);
        }
        
        return result;
    }
}
```

#### 3.3.2 限流设计

采用**令牌桶算法**实现接口限流：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          令牌桶限流算法                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│                    ┌─────────────────────┐                              │
│                    │    令牌桶 (Bucket)   │                              │
│                    │   容量: 10 令牌      │                              │
│                    │   当前: 7 令牌       │                              │
│                    └──────────┬──────────┘                              │
│                               │                                         │
│         ┌─────────────────────┼─────────────────────┐                   │
│         │                     │                     │                   │
│         ▼                     ▼                     ▼                   │
│  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐             │
│  │ 定时补充令牌 │      │  请求到达   │      │  请求处理   │             │
│  │  2个/秒     │      │  消耗1令牌  │      │  正常响应   │             │
│  └─────────────┘      └─────────────┘      └─────────────┘             │
│                              │                                         │
│                              ▼                                         │
│                       ┌─────────────┐                                  │
│                       │ 令牌不足    │                                  │

```

**限流配置**：

| 接口类型 | 限流策略 | 说明 |
|----------|----------|------|
| 登录接口 | 2次/秒/IP | 防止暴力破解 |
| 查询接口 | 10次/秒/用户 | 防止接口滥用 |
| 控制接口 | 5次/秒/用户 | 保护设备稳定性 |

## 5. 总结与建议

### 5.1 技术选型总结

本项目采用的技术栈已在第3.2节详细说明，核心组合如下：

| 层级 | 技术选型 | 核心优势 |
|------|----------|----------|
| 后端框架 | Spring Boot 3.2.3 + Java 21 | 企业级特性完善，性能优异 |
| 数据存储 | PostgreSQL + Redis | 功能强大，性能优秀 |
| 消息中间件 | MQTT + Kafka | IoT适配，高吞吐 |
| 前端框架 | Vue.js 3 + Vite | 开发效率高，性能优秀 |
| 移动端 | UniApp | 一套代码多端发布 |
| 监控运维 | Prometheus + Grafana | 开源免费，功能完善 |
| 容器化 | Docker + Docker Compose | 环境一致性，部署简单 |

详细的技术选型理由和对比分析请参见第3.2节。

### 5.2 架构设计总结

1. **分层架构**：采用经典的四层架构（Controller → Service → Repository → Entity），职责清晰
2. **微服务就绪**：服务无状态设计，支持水平扩展和微服务拆分
3. **实时通信**：MQTT + WebSocket双协议支持，满足IoT设备通信和前端实时推送需求
4. **安全可靠**：JWT认证 + RBAC权限 + 限流保护，多层次安全保障
5. **可观测性**：完善的监控告警体系，支持问题快速定位

### 5.3 后续演进建议

1. **短期优化**：
   - 完善单元测试和集成测试覆盖率
   - 优化数据库查询性能，添加必要索引
   - 完善API文档和开发文档

2. **中期演进**：
   - 引入服务网格（Istio）支持微服务治理
   - 实现数据库读写分离和分库分表
   - 引入ELK日志分析平台

3. **长期规划**：
   - 支持多云部署，提高可用性
   - 引入AI/ML平台，增强智能分析能力
   - 支持更多IoT设备协议（CoAP、LwM2M等）
