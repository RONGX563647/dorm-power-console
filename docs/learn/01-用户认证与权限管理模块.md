## 二、需求分析与设计

### 2.1 业务场景

**场景 1：用户登录**
```
用户输入用户名密码 → 系统验证 → 返回 Token → 用户后续请求携带 Token
```

**场景 2：权限控制**
```
用户访问资源 → 系统检查 Token → 验证权限 → 允许/拒绝访问
```

**场景 3：角色管理**
```
管理员创建角色 → 分配权限 → 给用户分配角色 → 用户继承角色权限
```

### 2.2 功能需求清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户登录 | 用户名密码认证，返回 JWT Token | P0 |
| 用户登出 | 使 Token 失效 | P0 |

| 用户管理 | 用户 CRUD、重置密码 | P0 |

| IP 访问控制 | IP 白名单/黑名单 | P2 |

### 2.3 技术选型与理由

| 技术 | 说明 | 版本 | 选型理由 |
|------|------|------|----------|
| **Spring Security** | 安全框架 | 3.2.x | ✅ **行业标准**：Spring 官方推荐的安全框架，生态完善<br>✅ **功能全面**：支持认证、授权、CSRF 防护、会话管理<br>✅ **灵活扩展**：支持自定义认证逻辑、权限表达式<br>✅ **学习价值**：企业级项目必备技能，就业市场需求大 |
| **JWT (jjwt)** | Token 认证 | 0.12.x | ✅ **无状态**：服务端不存储 Token，降低服务器压力<br>✅ **跨域支持**：适合前后端分离架构<br>✅ **自包含**：Token 中包含用户信息，减少数据库查询<br>✅ **安全性**：支持签名验证，防止篡改<br>⚠️ **注意**：实际项目中使用 String 作为主键而非 UUID，便于分布式 ID 生成 |
| **BCrypt** | 密码加密 | - | ✅ **加盐哈希**：自动处理盐值，无需手动管理<br>✅ **自适应**：可调整加密强度（work factor）<br>✅ **不可逆**：即使数据库泄露也无法还原密码<br>✅ **Spring 集成**：Spring Security 内置支持 |
| **Vue Router** | 路由守卫 | 4.x | ✅ **官方路由**：Vue.js 官方维护，兼容性好<br>✅ **路由守卫**：实现前端权限控制（beforeEach）<br>✅ **动态路由**：支持根据权限动态加载路由<br>✅ **TypeScript**：完整的类型支持 |

**技术对比与取舍**：
- **Session vs JWT**：选择 JWT 因为项目是前后端分离架构，且需要支持移动端
- **MD5 vs BCrypt**：选择 BCrypt 因为更安全，支持自适应加密强度
- **Shiro vs Spring Security**：选择 Spring Security 因为生态更完善，与 Spring Boot 集成更好

```java
import jakarta.persistence.*;
import lombok.Data;
import java.time.LocalDateTime;
import java.util.Set;

@Data
@Entity
@Table(name = "user_account")
public class UserAccount {
    
    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    private UUID id;
    
    @Column(unique = true, nullable = false, length = 50)
    private String username;
    
    @Column(nullable = false)

    @Column(length = 100)
    private String email;
    
    @Column(length = 20)

    private boolean accountNonExpired = true;
    private boolean accountNonLocked = true;
    private boolean credentialsNonExpired = true;
    private boolean enabled = true;
    
    private LocalDateTime createdAt = LocalDateTime.now();
    private LocalDateTime updatedAt = LocalDateTime.now();
    private LocalDateTime lastLoginAt;
    
    @ManyToMany(fetch = FetchType.EAGER)
    @JoinTable(
        name = "user_role",
        joinColumns = @JoinColumn(name = "user_id"),
        inverseJoinColumns = @JoinColumn(name = "role_id")
    )
    private Set<Role> roles;
}
```

```java
package com.dormpower.dto.request;

import io.swagger.v3.oas.annotations.media.Schema;
import jakarta.validation.constraints.NotBlank;
import jakarta.validation.constraints.Size;

/**
 * 登录请求DTO
 * 
 * 设计说明:
 * 1. 使用 @Schema 添加 API 文档注解
 * 2. 使用 @NotBlank 和 @Size 进行数据验证
 * 3. 不包含敏感操作(如密码加密),仅作为数据传输载体
 */
@Schema(description = "登录请求")
public class LoginRequest {

    @NotBlank(message = "用户名不能为空")
    @Size(min = 3, max = 50, message = "用户名长度必须在3-50个字符之间")
    @Schema(description = "用户名", example = "admin", required = true, minLength = 3, maxLength = 50)
    private String username;

    @NotBlank(message = "密码不能为空")
    @Size(min = 6, max = 100, message = "密码长度必须在6-100个字符之间")
    @Schema(description = "密码", example = "admin123", required = true, minLength = 6, maxLength = 100)
    private String password;

    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }
}
```

```java
package com.dormpower.dto.response;

import io.swagger.v3.oas.annotations.media.Schema;

import java.util.UUID;

/**
 * 认证响应DTO
 * 
 * 设计说明:
 * 1. 不返回密码等敏感信息
 * 2. 包含Token和用户基本信息
 * 3. 使用 Builder 模式构建对象
 */
@Schema(description = "认证响应")
public class AuthResponse {

    @Schema(description = "访问令牌")
    private String accessToken;

    @Schema(description = "刷新令牌")
    private String refreshToken;

    @Schema(description = "用户ID")
    private UUID userId;

    @Schema(description = "用户名")
    private String username;

    @Schema(description = "邮箱")
    private String email;

    @Schema(description = "Token类型", example = "Bearer")
    private String tokenType = "Bearer";

    @Schema(description = "Token过期时间(秒)", example = "3600")
    private Long expiresIn;

    public AuthResponse() {
    }

    private AuthResponse(String accessToken, String refreshToken, UUID userId, 
                         String username, String email, Long expiresIn) {
        this.accessToken = accessToken;
        this.refreshToken = refreshToken;
        this.userId = userId;
        this.username = username;
        this.email = email;
        this.expiresIn = expiresIn;
    }

    public static Builder builder() {
        return new Builder();
    }

    public String getAccessToken() {
        return accessToken;
    }

    public void setAccessToken(String accessToken) {
        this.accessToken = accessToken;
    }

    public String getRefreshToken() {
        return refreshToken;
    }

    public void setRefreshToken(String refreshToken) {
        this.refreshToken = refreshToken;
    }

    public UUID getUserId() {
        return userId;
    }

    public void setUserId(UUID userId) {
        this.userId = userId;
    }

    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getEmail() {
        return email;
    }

    public void setEmail(String email) {
        this.email = email;
    }

    public String getTokenType() {
        return tokenType;
    }

    public void setTokenType(String tokenType) {
        this.tokenType = tokenType;
    }

    public Long getExpiresIn() {
        return expiresIn;
    }

    public void setExpiresIn(Long expiresIn) {
        this.expiresIn = expiresIn;
    }

    public static class Builder {
        private String accessToken;
        private String refreshToken;
        private UUID userId;
        private String username;
        private String email;
        private Long expiresIn;

        public Builder accessToken(String accessToken) {
            this.accessToken = accessToken;
            return this;
        }

        public Builder refreshToken(String refreshToken) {
            this.refreshToken = refreshToken;
            return this;
        }

        public Builder userId(UUID userId) {
            this.userId = userId;
            return this;
        }

        public Builder username(String username) {
            this.username = username;
            return this;
        }

        public Builder email(String email) {
            this.email = email;
            return this;
        }

        public Builder expiresIn(Long expiresIn) {
            this.expiresIn = expiresIn;
            return this;
        }

        public AuthResponse build() {
            return new AuthResponse(accessToken, refreshToken, userId, 
                                   username, email, expiresIn);
        }
    }
}
```

**DTO 设计原则**:

```
┌─────────────────────────────────────────────┐
│            DTO 设计最佳实践                   │
├─────────────────────────────────────────────┤
│ 1. 职责单一:                                 │
│    - Request DTO: 接收前端数据,添加验证       │
│    - Response DTO: 返回前端数据,过滤敏感信息  │
│                                            │
│ 2. 命名规范:                                 │
│    - 请求DTO: XxxRequest.java                │
│    - 响应DTO: XxxResponse.java               │
│    - 消息DTO: XxxMessage.java                │
│                                            │
│ 3. 验证注解:                                 │
│    - @NotNull: 不能为null                    │
│    - @NotBlank: 不能为空白字符串              │
│    - @Size: 字符串/集合大小限制               │
│    - @Email: 邮箱格式验证                    │
│    - @Min/@Max: 数值范围验证                 │
│                                            │
│ 4. API文档:                                  │
│    - @Schema: OpenAPI 3.0 文档注解           │
│    - description: 字段说明                   │
│    - example: 示例值                         │
│    - required: 是否必填                      │
│                                            │
│ 5. 与Entity的区别:                           │
│    - Entity: 数据库映射,包含JPA注解           │
│    - DTO: 数据传输,包含验证注解               │
│    - 两者可以相互转换,使用Mapper或手动转换    │
└─────────────────────────────────────────────┘
```

```java
package com.dormpower.service.impl;

import com.dormpower.dto.request.LoginRequest;
import com.dormpower.dto.response.AuthResponse;
import com.dormpower.model.UserAccount;
import com.dormpower.repository.UserAccountRepository;
import com.dormpower.service.AuthService;
import com.dormpower.utils.JwtTokenProvider;
import lombok.RequiredArgsConstructor;
import org.springframework.security.authentication.AuthenticationManager;
import org.springframework.security.authentication.UsernamePasswordAuthenticationToken;
import org.springframework.security.core.Authentication;
import org.springframework.stereotype.Service;

@Service
@RequiredArgsConstructor
public class AuthServiceImpl implements AuthService {
    
    private final AuthenticationManager authenticationManager;
    private final UserAccountRepository userAccountRepository;
    private final JwtTokenProvider jwtTokenProvider;
    
    @Override
    public AuthResponse login(LoginRequest request) {
        // 1. 认证
        Authentication authentication = authenticationManager.authenticate(
            new UsernamePasswordAuthenticationToken(
                request.getUsername(),
                request.getPassword()
            )
        );
        
        // 2. 生成 Token
        String accessToken = jwtTokenProvider.generateAccessToken(authentication);
        String refreshToken = jwtTokenProvider.generateRefreshToken(authentication);
        
        // 3. 更新最后登录时间
        UserAccount user = userAccountRepository.findByUsername(request.getUsername()).orElseThrow();
        user.setLastLoginAt(java.time.LocalDateTime.now());
        userAccountRepository.save(user);
        
        // 4. 返回响应
        return AuthResponse.builder()
            .accessToken(accessToken)
            .refreshToken(refreshToken)
            .userId(user.getId())
            .username(user.getUsername())
            .build();
    }
    
    @Override
    public void logout(String token) {
        // 将 Token 加入黑名单
        jwtTokenProvider.addToBlacklist(token);
    }
    
    @Override
    public AuthResponse refreshToken(String refreshToken) {
        // 验证刷新 Token
        if (!jwtTokenProvider.validateToken(refreshToken)) {
            throw new RuntimeException("Invalid refresh token");
        }
        
        String username = jwtTokenProvider.getUsernameFromToken(refreshToken);
        UserAccount user = userAccountRepository.findByUsername(username).orElseThrow();
        
        // 生成新 Token
        Authentication authentication = jwtTokenProvider.getAuthentication(refreshToken);
        String newAccessToken = jwtTokenProvider.generateAccessToken(authentication);
        
        return AuthResponse.builder()
            .accessToken(newAccessToken)
            .refreshToken(refreshToken)
            .userId(user.getId())
            .username(user.getUsername())
            .build();
    }
    
    @Override
    public UserAccount getCurrentUser() {
        // 从安全上下文获取当前用户
        String username = jwtTokenProvider.getCurrentUserUsername();
        return userAccountRepository.findByUsername(username).orElseThrow();
    }
}
```

```typescript
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';
import type { User, AuthState } from '@/types/auth';

export const useAuthStore = defineStore('auth', () => {

    user.value = newUser;
  }
  
  // 加载用户信息

      const userData = await authApi.getCurrentUser();

  // 登出

    if (token.value) {
      await authApi.logout(token.value);
    }
    token.value = null;
    user.value = null;
    localStorage.removeItem('token');
  }
  
  return {
    token,
    user,
    isAuthenticated,
    roles,
    permissions,
    setToken,
```

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {

            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
```

#### 问题 2：Token 刷新时序问题
**现象**: 多个请求同时触发 Token 刷新

**解决方案**:
```typescript
// 在 API 拦截器中控制
let isRefreshing = false;
let refreshSubscribers: Array<(token: string) => void> = [];

axios.interceptors.response.use(
  response => response,
  async error => {
    const { config, response } = error;
    
    if (response?.status === 401 && !config._retry) {
      if (isRefreshing) {
        // 等待刷新完成
        return new Promise(resolve => {
          refreshSubscribers.push(token => {
            config.headers.Authorization = `Bearer ${token}`;
            resolve(axios(config));
          });
        });
      }
      
      config._retry = true;
      isRefreshing = true;
      
      try {
        const newToken = await refreshToken();
        refreshSubscribers.forEach(cb => cb(newToken));
        return axios(config);
      } finally {
        isRefreshing = false;
        refreshSubscribers = [];
      }
    }
    
    return Promise.reject(error);
  }
);
```
