## 🚀 使用方法

### 本地开发

1. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

2. **启动开发服务器**
   
   Windows PowerShell:
   ```powershell
   .\serve.ps1
   ```
   
   Linux/Mac:
   ```bash
   ./serve.sh
   ```

3. **访问预览**
   
   打开浏览器访问：http://localhost:8000

### 构建静态站点

```bash
.\build.ps1  # Windows
./build.sh   # Linux/Mac
```

构建产物在 `../../site` 目录。

### 部署到 GitHub Pages

1. **修改 mkdocs.yml 中的配置**
   ```yaml
   site_url: https://your-github-username.github.io/dorm-power-console/
   repo_url: https://github.com/your-github-username/dorm-power-console
   ```

2. **部署**
   ```bash
   .\deploy.ps1  # Windows
   ./deploy.sh   # Linux/Mac
   ```

3. **自动部署**
   
   推送到 main 分支会自动触发 GitHub Actions 部署。

## ⚠️ 注意事项

### 1. 文件命名

- 所有 Markdown 文件使用 UTF-8 编码
- 文件名使用中文，避免特殊字符
- 保持文件名简洁明了

### 2. 代码块格式

- 必须指定语言标识（如 ```java）
- 确保代码块正确闭合
- 避免在代码块中使用未转义的特殊字符

### 3. 图片资源

- 图片放在 `assets/` 目录
- 使用相对路径引用
- 优化图片大小

### 4. 链接

- 使用相对路径链接其他文档
- 确保链接目标文件存在
- 避免死链

## 🔧 配置说明

### mkdocs.yml 主要配置项

```yaml
# 站点基本信息
site_name: 站点名称
site_description: 站点描述
site_author: 作者
site_url: 站点 URL

# 主题配置
theme:
  name: material
  language: zh
  features:
    - navigation.tabs  # 顶部导航
    - content.code.copy  # 代码复制

# Markdown 扩展
markdown_extensions:
  - admonition  # 提示框
  - pymdownx.emoji  # Emoji
  - pymdownx.highlight  # 代码高亮

# 插件
plugins:
  - search  # 搜索
  - minify  # 压缩
```

## 🐛 故障排除

### 1. 构建失败

**错误**: `Config value 'docs_dir': The path 'docs' isn't an existing directory.`

**解决**: 确保 `docs_dir` 配置正确，设置为 `.` 表示当前目录。

### 2. 代码高亮失效

**错误**: 代码块没有高亮

**解决**: 检查是否指定了语言标识，如 ```java。

### 3. 中文乱码

**错误**: 页面显示乱码

**解决**: 确保文件保存为 UTF-8 编码。

### 4. 部署失败

**错误**: GitHub Pages 部署失败

**解决**: 
- 检查 GitHub Actions 日志
- 确认 permissions 配置正确
- 检查 site_dir 路径
