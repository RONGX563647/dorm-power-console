# 后端教学设计总览

> 本教学设计面向“基于真实项目带学生理解 Spring Boot 后端”的场景，强调代码走读、架构理解、接口联调与安全意识同步推进。

---

## 1. 教学定位

- **教学对象**：具备 Java 基础、了解 HTTP 与数据库基本概念的学习者
- **项目载体**：宿舍电力管理系统后端，核心目录位于 `backend/`
- **教学目标**：让学生不仅知道“Spring Boot 能做什么”，还知道“这个项目为什么这样分层、为什么这样接入 MQTT、为什么这样做安全控制”
- **教学方式**：理论讲解 + 代码走读 + 接口验证 + 课后小练习

---

## 2. 教学总目标

完成本单元后，学生应能够：

1. 说清楚该项目的整体技术栈、分层结构和请求处理链路。
2. 读懂 `controller`、`service`、`repository`、`model` 的职责边界。
3. 理解 Spring Security、JWT、AOP、限流、缓存在项目中的作用与配合方式。
4. 能对照真实代码定位认证、鉴权、异常处理、WebSocket、MQTT 等关键实现。
5. 能在现有项目基础上进行小范围扩展，如新增接口、添加限流注解、补充缓存或安全校验。

---

## 3. 教学模块划分

### 模块一：项目架构与技术栈

- 对应文档：`01_项目架构与技术栈详解.md`
- 核心问题：项目由哪些模块组成？请求是怎么走到数据库和设备侧的？
- 重点内容：Spring Boot 启动、包结构、配置文件、多环境配置、OpenAPI、MQTT、WebSocket
- 课堂产出：学生能根据目录快速说出一个功能应放在哪一层实现

### 模块二：核心中间件与安全机制

- 对应文档：`02_核心中间件与安全机制.md`
- 核心问题：项目如何保证接口安全、如何统一记录日志、如何限制恶意请求、如何提升查询性能？
- 重点内容：Security 过滤器链、JWT、AOP、限流、缓存、异常处理
- 课堂产出：学生能画出“请求进入系统后的安全与中间件处理链”

### 模块三：业务逻辑与数据层设计

- 对应文档：`03_业务逻辑与数据层设计.md`
- 核心问题：Controller、Service、Repository、Entity 四层如何分工？事务为什么放在服务层？
- 重点内容：控制器职责、服务编排、JPA 实体建模、方法名查询、JPQL、事务边界
- 课堂产出：学生能顺着真实代码走完一条业务链路，并判断逻辑该落在哪一层

### 模块四：设备通信与实时交互设计

- 对应文档：`04_设备通信与实时交互设计.md`
- 核心问题：设备状态如何进入后端，后端如何把变化实时推送到前端？
- 重点内容：MQTT 主题设计、`MqttBridge` 消息分发、WebSocket 订阅模型、定时任务补偿机制
- 课堂产出：学生能画出 telemetry 消息从设备到前端的完整链路

### 模块五：监控、告警与系统运维设计

- 对应文档：`05_监控告警与系统运维设计.md`
- 核心问题：系统如何监控自身状态、生成告警、沉淀日志并完成备份清理？
- 重点内容：监控指标采集、告警阈值、通知中心、系统日志、备份与清理、运维任务调度
- 课堂产出：学生能理解一个后端系统如何形成“监控-告警-通知-处置”的运维闭环

## 4. 建议课时安排

| 课次    | 时长   | 主题                 | 课堂活动                                                                             |
| ------- | ------ | -------------------- | ------------------------------------------------------------------------------------ |
| 第 1 次 | 2 学时 | 项目全景与技术栈     | 讲解目录、启动项目、对照 `pom.xml` 和 `application.yml`                              |
| 第 2 次 | 2 学时 | 安全认证链路         | 走读 `SecurityConfig`、`JwtAuthenticationFilter`、`JwtUtil`                          |
| 第 3 次 | 2 学时 | 中间件横切能力       | 走读 `ApiAspect`、`RateLimit`、`GlobalExceptionHandler`                              |
| 第 4 次 | 2 学时 | 业务逻辑与数据层     | 走读 `DeviceService`、`TelemetryService`、`Role`、`Permission`、`UserRole`           |
| 第 5 次 | 2 学时 | 实时通信与状态同步   | 走读 `MqttBridge`、`WebSocketHandler`、`WebSocketManager`、定时任务类                |
| 第 6 次 | 2 学时 | 监控、告警与运维闭环 | 走读 `MonitoringService`、`AlertService`、`NotificationService`、`DataBackupService` |

## 5. 每次课的基本结构

1. **问题导入（10 分钟）**：先说明本节想解决什么工程问题。
2. **核心概念（20 分钟）**：只讲理解代码所需的最小理论。
3. **代码走读（35 分钟）**：从入口类或配置类出发，顺着调用链往下走。
4. **接口验证（15 分钟）**：用 Swagger 或前端请求验证讲解内容。
5. **课堂练习（10 分钟）**：做一个局部修改或定位一个问题。

---

## 6. 教学重点与难点

### 教学重点

- Spring Boot 项目的分层设计与职责边界
- 安全过滤器链与 JWT 鉴权流程
- AOP 在日志、审计、限流中的实际用途
- MQTT 与 WebSocket 在设备系统中的分工
- 事务边界、实体关系与仓储查询在真实业务中的落地
- 监控、告警、日志、备份如何组成运维闭环

### 教学难点

- 学生容易把“认证”和“授权”混为一谈
- 学生容易把 Controller 写成“大杂烩”而不是让 Service 承担业务逻辑
- 学生容易只会用注解，不理解背后的过滤器链、切面或缓存管理器
- 学生容易忽略项目中的“当前实现”和“理想改进方向”之间的差异
- 学生容易把“实时通信”“定时轮询”“系统监控”混成同一个概念
- 学生容易忽略运维能力也是后端系统设计的一部分

## 7. 建议课堂实践任务

1. 说明一次 `/api/auth/login` 请求依次经过了哪些类和机制。
2. 给某个新接口加上 `@RateLimit` 和 `@AuditLog`，观察效果。
3. 在 `DeviceService` 或 `TelemetryService` 中解释缓存命中与失效条件。
4. 分析 `MqttBridge` 收到设备状态后，数据如何落库并推送到前端。
5. 根据 `GlobalExceptionHandler` 说明系统如何把异常转成统一 HTTP 响应。
6. 找出一个使用 `@Transactional` 的服务方法，并解释为什么事务边界放在这里。
7. 结合 `MonitoringService`、`AlertService` 和 `NotificationService` 画出告警流转图。

---

## 8. 课后考核建议

- **基础题**：解释 `@SpringBootApplication`、`@RestControllerAdvice`、`@Cacheable` 的作用。
- **理解题**：画出 JWT 鉴权流程图。
- **实践题**：新增一个受保护接口，并补充认证校验与限流。
- **提升题**：指出当前实现中仍可改进的安全点，例如 WebSocket 跨域、Token 黑名单持久化、Session 策略显式化等。
- **综合题**：说明设备 telemetry 如何触发告警并通过 WebSocket 或通知中心反馈到管理端。

---

## 9. 教学使用建议

- 讲解时以真实代码为准，不把项目中尚未实现的方案讲成“当前已上线方案”。
- 对安全、缓存、限流等内容，区分“当前实现”和“推荐演进方向”。
- 每章结束后要求学生回到代码目录中做一次定位练习，强化工程感。

---

## 10. 文档阅读顺序

1. 先读 `01_项目架构与技术栈详解.md`，建立全局认识。
2. 再读 `02_核心中间件与安全机制.md`，理解系统如何保证安全和稳定。
3. 接着读 `03_业务逻辑与数据层设计.md`，建立分层与事务的工程认知。
4. 然后读 `04_设备通信与实时交互设计.md`，理解设备系统的实时链路。
5. 最后读 `05_监控告警与系统运维设计.md`，把系统维护能力补齐。
