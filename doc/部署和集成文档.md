# 宿舍电源管理系统部署和集成文档

## 1. 部署概述

本部署和集成文档描述了宿舍电源管理系统的部署方法，包括本地开发环境部署和Docker容器化部署。系统由后端和前端两部分组成，需要分别部署。

### 1.1 系统组件

- **后端**：基于FastAPI框架的Python服务
- **前端**：基于Next.js框架的React应用
- **数据库**：SQLite（开发环境）或PostgreSQL（生产环境）
- **MQTT Broker**：可选，用于与设备通信

## 2. 本地开发环境部署

### 2.1 后端部署

#### 2.1.1 环境要求

- Python 3.8+
- pip
- venv（Python虚拟环境）

#### 2.1.2 部署步骤

1. **克隆代码**：

```bash
git clone https://github.com/Evildoersudo/back_end.git
cd back_end
```

2. **创建虚拟环境**：

```bash
python -m venv .venv
```

3. **激活虚拟环境**：

- Windows：
  ```bash
  .venv\Scripts\activate
  ```

- macOS/Linux：
  ```bash
  source .venv/bin/activate
  ```

4. **安装依赖**：

```bash
pip install -r requirements.txt
```

5. **配置环境变量**：

```bash
cp .env.example .env
```

编辑 `.env` 文件，配置必要的环境变量：

```env
# 数据库配置
DATABASE_URL=sqlite:///./iot_backend.db

# MQTT配置
MQTT_ENABLED=0
MQTT_HOST=localhost
MQTT_PORT=1883
MQTT_TOPIC_PREFIX=dorm

# 管理员账号配置
ADMIN_USERNAME=admin
ADMIN_EMAIL=admin@dorm.local
ADMIN_PASSWORD=admin123

# 系统配置
ONLINE_TIMEOUT_SECONDS=10
CMD_TIMEOUT_SECONDS=30
```

6. **启动服务**：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

7. **验证服务**：

访问 `http://127.0.0.1:8000/health` 检查服务是否正常运行。

### 2.2 前端部署

#### 2.2.1 环境要求

- Node.js 18+
- npm 或 yarn

#### 2.2.2 部署步骤

1. **克隆代码**：

```bash
git clone https://github.com/Evildoersudo/dorm-power-console.git
cd dorm-power-console
```

2. **安装依赖**：

```bash
npm install
# 或
yarn install
```

3. **配置API地址**：

编辑 `src/lib/fetcher.ts` 文件，配置后端API地址：

```typescript
const BASE_URL = 'http://localhost:8000';
```

4. **启动开发服务器**：

```bash
npm run dev
# 或
yarn dev
```

5. **访问前端应用**：

访问 `http://localhost:3000` 查看前端应用。

## 3. Docker容器化部署

### 3.1 后端Docker部署

#### 3.1.1 环境要求

- Docker
- Docker Compose

#### 3.1.2 部署步骤

1. **编辑环境变量**：

编辑 `docker-compose.yml` 文件，配置必要的环境变量：

```yaml
version: '3.8'
services:
  backend:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql+psycopg://admin:password@db:5432/dorm_power
      - MQTT_ENABLED=1
      - MQTT_HOST=mqtt
      - MQTT_PORT=1883
      - MQTT_TOPIC_PREFIX=dorm
      - ADMIN_USERNAME=admin
      - ADMIN_EMAIL=admin@dorm.local
      - ADMIN_PASSWORD=admin123
      - ONLINE_TIMEOUT_SECONDS=10
      - CMD_TIMEOUT_SECONDS=30
    depends_on:
      - db
      - mqtt

  db:
    image: postgres:13
    environment:
      - POSTGRES_USER=admin
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=dorm_power
    volumes:
      - postgres_data:/var/lib/postgresql/data

  mqtt:
    image: emqx/emqx:4.4.3
    ports:
      - "1883:1883"
      - "18083:18083"

volumes:
  postgres_data:
```

2. **构建和启动容器**：

```bash
docker-compose up -d --build
```

3. **验证服务**：

访问 `http://localhost:8000/health` 检查服务是否正常运行。

### 3.2 前端Docker部署

#### 3.2.1 环境要求

- Docker
- Docker Compose

#### 3.2.2 部署步骤

1. **编辑Dockerfile**：

确保 `Dockerfile` 文件配置正确：

```dockerfile
FROM node:18-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM node:18-alpine AS production
WORKDIR /app
COPY --from=build /app/package*.json ./
COPY --from=build /app/.next ./.next
COPY --from=build /app/public ./public
RUN npm install --production
EXPOSE 3000
CMD ["npm", "start"]
```

2. **编辑docker-compose.yml**：

```yaml
version: '3.8'
services:
  frontend:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_API_URL=http://backend:8000
```

3. **构建和启动容器**：

```bash
docker-compose up -d --build
```

4. **访问前端应用**：

访问 `http://localhost:3000` 查看前端应用。

## 4. 集成配置

### 4.1 前后端集成

1. **后端配置**：

确保后端服务运行在指定端口（默认为8000）。

2. **前端配置**：

编辑 `src/lib/fetcher.ts` 文件，配置后端API地址：

```typescript
const BASE_URL = 'http://localhost:8000';
```

### 4.2 MQTT集成

1. **启用MQTT**：

编辑 `.env` 文件，设置 `MQTT_ENABLED=1`。

2. **配置MQTT Broker**：

设置 MQTT 主机、端口和主题前缀：

```env
MQTT_HOST=localhost
MQTT_PORT=1883
MQTT_TOPIC_PREFIX=dorm
```

3. **设备连接**：

设备需要按照以下主题格式与后端通信：

- 状态上报：`dorm/{room}/{device}/status`
- 遥测上报：`dorm/{room}/{device}/telemetry`
- 命令回执：`dorm/{room}/{device}/ack`
- 事件上报：`dorm/{room}/{device}/event`
- 离线通知：`dorm/{room}/{device}/lwt`
- 命令接收：`dorm/{room}/{device}/cmd`

### 4.3 数据库集成

#### 4.3.1 SQLite（开发环境）

- 默认配置：`DATABASE_URL=sqlite:///./iot_backend.db`
- 无需额外安装数据库服务

#### 4.3.2 PostgreSQL（生产环境）

- 配置示例：`DATABASE_URL=postgresql+psycopg://user:password@localhost:5432/dorm_power`
- 需要安装PostgreSQL驱动：`pip install psycopg[binary]`

## 5. 系统维护

```bash
git pull
```

2. **重新部署**：

- 本地环境：重启服务
- Docker环境：重新构建和启动容器

## 6. 故障排查

### 6.1 常见问题

#### 6.1.1 后端服务无法启动

- 检查端口是否被占用
- 检查环境变量配置是否正确
- 检查数据库连接是否正常

#### 6.1.2 前端无法连接后端

- 检查后端服务是否运行
- 检查前端API地址配置是否正确
- 检查网络连接是否正常

#### 6.1.3 设备无法连接MQTT

- 检查MQTT Broker是否运行
- 检查设备网络连接是否正常
- 检查MQTT主题格式是否正确

### 6.2 调试工具

- **健康检查**：`http://localhost:8000/health`
- **API文档**：`http://localhost:8000/docs`
- **MQTT调试**：使用MQTTX等工具测试MQTT连接

## 7. 性能优化

### 7.1 后端优化

- 使用Gunicorn作为生产服务器
- 启用数据库连接池
- 优化查询，添加适当的索引
- 对于高频数据采集，考虑使用消息队列

### 7.2 前端优化

- 启用代码分割
- 优化图片和资源
- 使用缓存策略
- 减少不必要的API请求

## 8. 安全考虑

### 8.1 后端安全

- 使用HTTPS加密传输
- 实现认证和授权机制
- 定期更新依赖库
- 限制数据库用户权限

### 8.2 前端安全

- 使用HTTPS
- 防止XSS攻击
- 防止CSRF攻击
- 安全存储用户凭证
