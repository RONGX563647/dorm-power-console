# H2数据库学习文档

## 一、H2数据库概述

### 1.1 什么是H2数据库

H2是一个开源的、轻量级的、纯Java编写的关系型数据库。它具有以下核心特点：

- **轻量级**：整个数据库仅约2MB的jar包
- **快速**：基于内存操作，性能优异
- **兼容性强**：支持标准SQL，兼容PostgreSQL、MySQL、Oracle等数据库语法
- **多模式运行**：支持内存模式、文件模式、服务器模式
- **零配置**：无需安装，开箱即用

### 1.2 H2 vs 其他嵌入式数据库对比

| 特性 | H2 | HSQLDB | Derby | SQLite |
|------|-----|--------|-------|--------|
| 纯Java | ✅ | ✅ | ✅ | ❌ |
| 内存模式 | ✅ | ✅ | ✅ | ❌ |
| PostgreSQL兼容 | ✅ | 部分 | ❌ | ❌ |
| 连接池支持 | ✅ | ✅ | ✅ | 有限 |
| Web控制台 | ✅ | ❌ | ❌ | ❌ |
| 性能 | 高 | 中 | 中 | 中 |

## 二、H2基础功能

### 2.1 运行模式详解

#### 2.1.1 内存模式（In-Memory）

数据完全存储在内存中，应用重启后数据丢失。

```yaml
# JDBC URL格式
jdbc:h2:mem:<databaseName>

# 项目中的实际配置（application-test.yml）
spring:
  datasource:
    url: jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE
    username: sa
    password: 
    driver-class-name: org.h2.Driver
```

**关键参数说明**：
- `DB_CLOSE_DELAY=-1`：数据库在最后一个连接关闭后保持打开状态（测试必需）
- `DB_CLOSE_ON_EXIT=FALSE`：JVM退出时不自动关闭数据库

#### 2.1.2 文件模式（Persistent）

数据持久化到磁盘文件。

```yaml
# JDBC URL格式
jdbc:h2:file:/path/to/database
jdbc:h2:./data/testdb  # 相对路径

# 示例配置
spring:
  datasource:
    url: jdbc:h2:file:./data/dorm_power;AUTO_SERVER=TRUE
```

**关键参数**：
- `AUTO_SERVER=TRUE`：允许多进程访问同一数据库文件

#### 2.1.3 服务器模式（Server Mode）

作为独立数据库服务器运行，支持多客户端连接。

```bash
# 启动H2服务器
java -jar h2.jar -tcp

# JDBC连接
jdbc:h2:tcp://localhost/~/test
jdbc:h2:tcp://192.168.1.100:9092/~/test
```

### 2.2 H2 Console（Web管理控制台）

H2内置了强大的Web管理界面：

```yaml
# 启用H2 Console（开发环境）
spring:
  h2:
    console:
      enabled: true
      path: /h2-console
      settings:
        web-allow-others: true  # 允许远程访问
```

**访问方式**：
- 地址：`http://localhost:8000/h2-console`
- 默认用户名：`sa`
- 默认密码：空

### 2.3 SQL兼容模式

H2支持多种数据库的兼容模式：

```yaml
# PostgreSQL兼容模式
spring:
  datasource:
    url: jdbc:h2:mem:testdb;MODE=PostgreSQL;DB_CLOSE_DELAY=-1
  jpa:
    properties:
      hibernate:
        dialect: org.hibernate.dialect.H2Dialect
```

**支持的兼容模式**：
- `MODE=PostgreSQL`：PostgreSQL语法兼容
- `MODE=MySQL`：MySQL语法兼容
- `MODE=Oracle`：Oracle语法兼容
- `MODE=SQLServer`：SQL Server语法兼容

### 3.2 内存存储机制

#### 3.2.1 Page Store架构

H2使用Page Store作为底层存储引擎：

```java
// Page类型
public abstract class Page {
    public static final int TYPE_DATA_LEAF = 0;      // 数据叶子节点
    public static final int TYPE_DATA_NODE = 1;      // 数据内部节点
    public static final int TYPE_DATA_OVERFLOW = 2;  // 溢出页
    public static final int TYPE_BTREE_LEAF = 3;     // B树叶子节点
    public static final int TYPE_BTREE_NODE = 4;     // B树内部节点
}
```

### 3.3 事务与并发控制

#### 3.3.1 MVCC（多版本并发控制）

H2采用MVCC实现事务隔离：

```sql
-- H2的事务隔离级别支持

-- READ_COMMITTED（默认）
```

## 四、场景使用指南

### 4.1 单元测试场景

#### 4.1.1 基本配置

```yaml
# application-test.yml
spring:
  datasource:
    url: jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE
    username: sa
    password: 
    driver-class-name: org.h2.Driver
  jpa:
    hibernate:
      ddl-auto: create-drop  # 测试结束自动删除表
    show-sql: true
    properties:
      hibernate:
        dialect: org.hibernate.dialect.H2Dialect
```

#### 4.1.2 测试类示例

```java
@SpringBootTest
@AutoConfigureMockMvc
@ActiveProfiles("test")
public class DeviceControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Autowired
    private DeviceRepository deviceRepository;

    @BeforeEach
    void setUp() {
        deviceRepository.deleteAll();
    }

    @Test
    @DisplayName("注册新设备-成功")
    void testRegisterDevice_Success() throws Exception {
        String deviceJson = """
            {
                "id": "device_001",
                "name": "A1-301智能插座",
                "room": "A1-301"
            }
            """;

        mockMvc.perform(post("/api/devices/register")
                .contentType(MediaType.APPLICATION_JSON)
                .content(deviceJson))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.id").value("device_001"));
    }
}
```

### 5.1 配置最佳实践

```yaml
# 推荐的测试环境配置
spring:
  datasource:
    url: jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE;MODE=PostgreSQL
    username: sa
    password: 
    driver-class-name: org.h2.Driver
  jpa:
    hibernate:
      ddl-auto: create-drop
    show-sql: false  # 生产测试建议关闭
    properties:
      hibernate:
        dialect: org.hibernate.dialect.H2Dialect
        format_sql: true
  h2:
    console:
      enabled: false  # 测试环境建议禁用
```

### 5.2 性能优化建议

```yaml
spring:
  datasource:
    url: jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;CACHE_SIZE=65536;LOCK_MODE=0
    hikari:
      maximum-pool-size: 10
      minimum-idle: 2
```

**参数说明**：
- `CACHE_SIZE=65536`：设置缓存大小（单位KB）
- `LOCK_MODE=0`：禁用锁（仅适用于单线程测试）

## 七、面试高频问题

### Q1：H2数据库适合用在生产环境吗？

**回答要点**：
- H2主要用于开发测试环境，不推荐用于高并发生产环境
- 原因：内存模式数据易丢失、并发能力有限、缺乏企业级特性
- 适用场景：单元测试、原型开发、嵌入式应用、CI/CD流水线

### Q2：H2内存模式和文件模式的区别？

| 对比项 | 内存模式 | 文件模式 |
|--------|----------|----------|
| 数据持久化 | ❌ 不持久化 | ✅ 持久化到磁盘 |
| 性能 | 极高 | 较高 |
| 适用场景 | 单元测试 | 本地开发、小型应用 |
| 数据安全 | 低 | 中 |

### Q3：为什么测试环境选择H2而不是真实数据库？

**回答要点**：
1. **速度**：内存操作，测试执行快
2. **隔离性**：每个测试独立数据库，互不影响
3. **便捷性**：无需安装配置外部数据库
4. **CI友好**：CI/CD流水线无需额外数据库服务
5. **成本**：节省数据库服务器资源

### Q4：H2如何保证与PostgreSQL的兼容性？

```yaml
# 配置兼容模式
spring:
  datasource:
    url: jdbc:h2:mem:testdb;MODE=PostgreSQL
  jpa:
    properties:
      hibernate:
        dialect: org.hibernate.dialect.H2Dialect
```

**注意事项**：
- 大部分SQL语法兼容
- 部分PostgreSQL特有函数可能不支持
- 建议关键业务逻辑仍需在真实PostgreSQL上验证

### Q5：H2的DB_CLOSE_DELAY参数有什么作用？

```yaml
# 参数说明
jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1
```

**作用**：
- 默认情况下，最后一个连接关闭时数据库会被删除
- `DB_CLOSE_DELAY=-1`：数据库在JVM生命周期内保持打开
- 测试场景必需：避免测试过程中数据库意外关闭

## 八、总结

H2数据库作为一款轻量级嵌入式数据库，在Java生态中扮演着重要角色。它特别适合：

1. **单元测试**：快速、隔离、零配置
2. **原型开发**：快速验证想法
3. **学习SQL**：低门槛实践
4. **CI/CD**：无需外部依赖

掌握H2的使用，能够显著提升开发效率和测试质量。
