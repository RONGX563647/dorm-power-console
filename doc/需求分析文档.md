# 宿舍电源管理系统需求分析文档

## 1. 项目概述

宿舍电源管理系统是一个面向高校宿舍的智能电源监控和控制系统，旨在通过物联网技术实现对宿舍用电设备的实时监控、远程控制和智能分析，提高用电安全性和能源利用效率。

### 1.1 项目目标

- 实现对宿舍用电设备的实时监控和状态管理
- 支持远程控制设备电源开关
- 提供用电数据的历史记录和趋势分析
- 通过AI分析提供节能建议和异常用电检测
- 确保系统的可靠性、安全性和可扩展性

### 1.2 系统架构

系统采用前后端分离架构：
- 前端：基于Next.js框架的Web应用，提供用户界面
- 后端：基于FastAPI框架的RESTful API服务
- 设备端：基于ESP8266等单片机的智能插座设备
- 通信协议：HTTP/HTTPS用于前后端通信，MQTT用于后端与设备通信
- 数据存储：SQLite/PostgreSQL用于持久化存储

## 4. 业务场景

### 4.1 场景一：设备监控与管理

**场景描述**：宿舍管理员需要查看所有宿舍的用电设备状态，了解设备是否在线、用电情况等。

**流程**：
1. 管理员登录系统
2. 进入设备总览页面，查看所有设备列表
3. 点击某个设备，查看详细状态信息
4. 根据需要，对设备进行控制操作

**需求关联**：FR1.2、FR1.3、FR2.1

### 4.2 场景二：用电数据分析

**场景描述**：学生需要查看自己宿舍的用电历史数据，了解用电趋势，优化用电习惯。

**流程**：
1. 学生登录系统
2. 进入历史分析页面，选择宿舍和时间范围
3. 查看用电趋势图表
4. 查看AI分析报告，获取节能建议

**需求关联**：FR3.2、FR3.3

### 4.3 场景三：远程控制

**场景描述**：学生在离开宿舍后，发现忘记关闭某些设备，需要远程关闭。

**流程**：
1. 学生登录系统
2. 进入设备详情页面，查看设备状态
3. 点击关闭按钮，下发关闭命令
4. 查看命令执行状态，确认设备已关闭

**需求关联**：FR2.1、FR2.3

### 4.4 场景四：异常用电检测

**场景描述**：系统检测到某个宿舍用电异常，需要及时通知管理员。

**流程**：
1. 系统持续监控设备用电数据
2. 检测到异常用电情况（如功率突增）
3. 生成异常报告并通过WebSocket推送
4. 管理员收到通知，及时处理

**需求关联**：FR3.1、FR3.3、FR4.1

## 6. 范围限定

- 本系统仅针对高校宿舍场景设计
- 系统支持的设备类型主要为智能插座
- 系统不包含计费功能
- 系统不涉及个人隐私数据的收集和存储

## 8. 风险分析

| 风险点 | 影响 | 应对措施 |
|--------|------|----------|
| 设备网络不稳定 | 数据采集不及时，控制命令执行失败 | 实现设备重连机制，增加数据缓存 |
| 系统负载过高 | 响应时间变长，服务不稳定 | 优化数据库查询，增加缓存机制 |
| 安全漏洞 | 系统被攻击，数据泄露 | 定期安全审计，更新依赖库，使用HTTPS |
| 设备兼容性 | 不同设备类型的适配问题 | 标准化设备通信协议，提供设备SDK |

## 9. 总结

宿舍电源管理系统是一个集设备监控、远程控制、数据采集和智能分析于一体的物联网系统。通过本系统，用户可以实时了解宿舍用电情况，远程控制用电设备，获取节能建议，提高用电安全性和能源利用效率。系统采用现代化的技术栈，具有良好的可扩展性和可靠性，能够满足高校宿舍场景的需求。
